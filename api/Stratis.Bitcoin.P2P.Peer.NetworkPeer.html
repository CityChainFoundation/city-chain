Class NetworkPeer

Represents a counterparty of the node on the network. This is usually another node, but it can be a wallet, an analytical robot, or any other network client or server that understands the protocol.

The network peer is connected either inbound, if it was the counterparty that established the connection to our node's listener, or outbound, if our node was the one connecting to a remote server.

public class NetworkPeer : object, INetworkPeer, IDisposable
Remarks

All instances of this object must be disposed or disconnected. Disconnect(String, Exception) and disposing methods have the same functionality and the disconnecting method is provided only for better readability of the code.

It is safe to try to disconnect or dispose this object multiple times, only the first call will be processed.
