Remarks

First we check if the block exists in chained header tree, then it check the block store and if it wasn't found there the block will be scheduled for download. Given callback is called when the block is obtained. If obtaining the block fails the callback will be called with null.

InitializeAsync(ChainedHeader)

Set the tip of ConsensusManager, if the given chainTip is not equal to Tip then rewind consensus until a common header is found.

Declaration
public async Task InitializeAsync(ChainedHeader chainTip)
Parameters
Type Name Description
ChainedHeader chainTip

Last common header between chain repository and block store if it's available, if the store is not available it is the chain repository tip.
Returns
Type Description
Task
Remarks

If Stratis.Bitcoin.Consensus.ConsensusManager.blockStore is not null (block store is available) then all block headers in Stratis.Bitcoin.Consensus.ConsensusManager.chainedHeaderTree will be marked as their block data is available. If store is not available the ConsensusManager won't be able to serve blocks from disk, instead all block requests that are not in memory will be sent to the Stratis.Bitcoin.Consensus.ConsensusManager.blockPuller.
